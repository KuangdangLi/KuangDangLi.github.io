初识React

August 25, 2022

基础API介绍#

正常情况下，React项目需要同时引入React、ReactDOM两个库才能正常运行。 React用来创建React元素，ReactDOM用来渲染。

const App = React.createElement('div',{className='red'},n)
//
React.createElement(
  type,
  [props],
  [...children]
)

React.createElement创建并返回指定类型的新 React 元素。其中的类型参数既可以是标签名字符串（如 ‘div’ 或 ‘span’），也可以是 React 组件 类型 （class 组件或函数组件），或是 React fragment 类型。属性参数可以用来设置类名或绑定事件等。第三个参数接受其它数据。

ReactDOM.render(App,root)
//render(element, container[, callback])

在提供的 container 里渲染一个 React 元素，并返回对该组件的引用（或者针对无状态组件返回 null）。

如果 React 元素之前已经在 container 里渲染过，这将会对其执行更新操作，并仅会在必要时改变 DOM 以映射最新的 React 元素。

如果提供了可选的回调函数，该回调将在组件被渲染或更新之后被执行。

注意：

在 React 18 中，render 函数已被 createRoot 函数所取代。

使用基础API#

用往常的思路来构建一个简单的+1按钮。

const React = window.React//CDN引入的React会挂载在全局变量上
const ReactDOM = window.ReactDOM

let  n = 0;

const App = React.createElement('div',null,[n,
    React.createElement(
        'button',
        {onClick:()=>n+=1},
        +1
        )
])

ReactDOM.render(App,root)

这时候如果点击屏幕上出现的+1按钮，我们会惊讶地发现n并没有发生变化。为了弄清楚这个让人迷惑不解的问题，我们给onClick打上log:console.log(n)。

log正常工作并且正常打出了自加后的n，但是ui却并没有刷新。这与我们一贯的思维(使用Vue的思维)不太一样。

或许是因为React不会因为数据变化自动刷新？尝试加入手动刷新。

const App = React.createElement('div',null,[n,
    React.createElement(
        'button',
        {onClick:()=>{n+=1;
        ReactDOM.render(App,root)
        }},
        +1
        )
])

结果与我们的期望背道而驰，UI并没有与按钮点击行为同步。

事实上，这个问题不是一个React问题，更多地是一个原生js问题。React.createElement会返回一个虚拟DOM，本质上是一个对象。当我们用这个对象为App赋值时，这个对象中的属性值已经固定了，特别是对于一些像n一样会进行传值的简单类型。当我们点击按钮调用onClick时，n的值改变了，但是并不会影响到App。事实上我们点击按钮手动刷新的App与最初并没有什么改变，因此UI并不会刷新。

我们需要每次render时的App都会是一个经过重新计算赋值的对象。我们决定将App改造为一个会返回虚拟DOM的函数。

const App = ()=>React.createElement('div',null,[n,
    React.createElement(
        'button',
        {onClick:()=>{n+=1;
        ReactDOM.render(App,root)
        }},
        +1
        )
])

这时，页面上的n可以如我们期望那般刷新了。App成为一个返回值为React元素的函数，每次调用都会重新计算，这让数据更新成为了可能。onClick会调用render，这使得UI与数据的更新同步。

App = React.createElement('div',null,n)
App = ()=>React.createElement('div',null,n)

理解上述两段代码的不同。
