Js函数执行时机

May 1, 2022

首先明确函数通过函数名加()来调用执行，函数声明不会调用执行函数。先声明后执行，在不同的时间函数可能面临着不同的环境与变量，其执行效果也不同。

例子#

let a = 1
function fn(){
    console.log(a)
}

未调用函数，无结果

let a = 1
function fn(){
    console.log(a)
}
fn()

声明变量并赋值；声明函数；随后调用函数，打印出a的值1.

let a = 1
function fn(){
    console.log(a)
}
a=2
fn()

声明变量并赋值；声明函数；修改变量值；随后调用函数，打印出a的值为2.

let a = 1
function fn(){
    console.log(a)
}
fn()
a=2

声明变量并赋值；声明函数；随后调用，打印出a的值1；修改a的值。

let a = 1
function fn(){
    setTimeout(()=>{
        console.log(a)
    },0)
}
fn() 
a = 2

声明变量并赋值；声明函数；随后调用，函数中嵌套的函数setTimeout要求打印的操作在队列中的任务完成后再执行；修改a的值；队列任务完成，打印a的值。

let i = 0
for(i = 0; i<6; i++){
  setTimeout(()=>{
    console.log(i)
  },0)
}

首先声明全局变量i，并赋值为0，for循环执行6次，打印出6个6.这是为什么呢?
在上个例子中我们知道了setTimeout会要求其后接函数在队列任务完毕完后再执行。因此在每一次的for循环中，console并没有执行，而是要等整个循环完毕后再打印。for循环结束后，i完成最后一次自加，其值为6，console执行，打印出6个6.
如何打印出0、1、2、3、4、5？

for(let i = 0; i<6; i++){
 setTimeout(()=>{
console.log(i)
},0)
}//在for循环中用let声明，每次循环都会创建一个块级作用域。

let i = 0
for(i = 0; i<6; i++){
!function(i){  
setTimeout(()=>{
  console.log(i)
},0)
}(i)
}//立即执行函数

let i = 0
for(i = 0; i<6; i++){
setTimeout((i)=>{
  console.log(i)
},0,i)
}//用setTimeout的附加参数，将i再传给箭头函数以实现保留每次循环中i的值。

let i
for(i = 0; i<6; i++){
const x = i
setTimeout(()=>{
  console.log(x)
})
}//在每一次循环中将i的值传递给const声明的变量x。const声明的变量保持常量值，无法重复声明以及修改。
